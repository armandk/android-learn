Class Overview

This class provides methods to encrypt and decrypt byte streams and byte arrays. The algorithm used to encrypt and decrypt is AES with GCM and no padding. The easiest way to generate and save encryption keys is by using EncryptionUtil, but it is not required to do so.

public static byte[] decrypt (byte[] encryptedData, byte[] encryptionKey)

Decrypts the contents of the given byte array with the given encryption key.

Parameters
encryptedData The encrypted bytes.
encryptionKey The key used to decrypt the bytes. This must be the same key that was used to encrypt the bytes, or else an exception will be thrown. When the application no longer needs the encryption key it should clear the array with a call like: Arrays.fill(encryptionKey, (byte) 0x0);
Returns

public static void decryptFile (String encryptedFilePath, String decryptedFilePath, byte[] encryptionKey)

Decrypts a file and saves the decrypted contents to another file.

Parameters
encryptedFilePath The complete filepath to the encrypted file.
decryptedFilePath The complete filepath to where the decrypted file should be written. If there is already a file at this filepath it will be overwritten.
encryptionKey The key used to decrypt the file, and which can be used to decrypt the file later. It is recommended to use EncryptionUtil to generate the encryption key, but this is not required. The encryption key must be 16, 24, or 32 bytes. When the application no longer needs the encryption key it should clear the array with a call like: Arrays.fill(encryptionKey, (byte) 0x0);
Throws
IOException Indicates something went wrong with IO or decryption.

public static void decryptStream (InputStream inputStream, OutputStream outputStream, byte[] encryptionKey)

Decrypts the contents of a stream and passes the decrypted contents to another stream. This mehtod can be used anywhere streams are used. For example, a file can be decrypted by passing this method a FileInputStream and a FileOutputStream. This method does not close any streams, that is the responsibility of the caller.

Parameters
inputStream The stream where encrypted bytes will be read from. This stream will be closed after calling this method.
outputStream The stream where decrypted bytes will be written to. This stream will be closed after calling this method.
encryptionKey The key used to decrypt the bytes. This must be the same key that was used to encrypt the bytes, or else an exception will be thrown. When the application no longer needs the encryption key it should clear the array with a call like: Arrays.fill(encryptionKey, (byte) 0x0);
Throws
IOException Indicates something went wrong with IO or decryption.
