      • getPreviousUserId

         final String getPreviousUserId()

        Retrieves the previous user id. For the very first onboarding flow, this will be 'null', then for later onboarding flows (create new account, or forgot passcode flows), this might be the same as that returned with getCurrentUserId if no user switch happens, or another user id if another user signs in or created.

        This information will be available after FlowStateListener.onUserSwitched, then in callback functions like FlowStateListener.onOfflineEncryptionKeyReady (if com.sap.cloud.mobile.foundation.settings.SharedDeviceService) was added into com.sap.cloud.mobile.foundation.mobileservices.SDKInitializer) and FlowStateListener.onFlowFinished, it can be used to compare with getCurrentUserId to know whether a user switch happened or not.

      • getMultipleUserMode

         final Boolean getMultipleUserMode()

        Single or multiple user model, for now the default is false. The main differences between the 2 modes are: 1, single mode supports biometric unlock while multiple user mode does not; 2, single model respects the policy from server side if not enabled, while multiple user model will use a default passcode policy if server disables it.

        Only the setting with the very first user onboarding flow will take effective. Once a user is onboarded , this setting will be saved into the local database, all flows afterwards will use the same setting from the database and ignore the one inside 'flowContext'.

      • getFlowRequestCode

         final Short getFlowRequestCode()

        For cases that the client code wants to use a single activity to start different flows and be able to identify which flow finishes in 'onActivityResult', this variable can be specified.

        Once a flow finishes, this setting will be restored to the default one, which is FlowConstants.FLOW_ACTIVITY_REQUEST_CODE
