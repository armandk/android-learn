      • onActivityResult

         Unit onActivityResult(Integer requestCode, Integer resultCode, Intent data)

        Inside this fragment, it may start the onbaroding flow then need to wait for the result. If onboarding flow finishes with 'OK', then finish this flow; otherwise, it means onboarding was cancelled at some point, in this case, the 'flowContext' in FlowContextRegistry may contain the 'current user' if was cancelled after authentication.

        When onboarding started from this screen gets back with 'cancel' result, in most cases, we can just ignore the current user in the flow context of onboarding, but if onboarding with a different user than the current one of sign in who is already onboarded, sign in screen needs to switch data to that user to let him/her input passcode to sign in.
