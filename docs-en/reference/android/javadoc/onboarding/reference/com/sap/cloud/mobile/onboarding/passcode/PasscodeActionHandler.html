PasscodeActionHandler

implements ActionHandler
com.sap.cloud.mobile.onboarding.passcode.PasscodeActionHandler

Class Overview

Callback interface to be implemented by the app developer using the passcode related flows. App developers have to provide the functionality accessible from within the passcode related screens by implementing the methods of this callback interface.

public abstract PasscodePolicy getPasscodePolicy (Fragment fragment)

Acquires the passcode policy from an arbitrary source. Most implementations are going to be synchronous (i.e. will invoke the listener right away), however there's possibility to create an asynchronous implementation that can, for example, download the policy from a server or call another application on the device.

public abstract void shouldTryPasscode (char[] passcode, PasscodeInputMode mode, Fragment fragment)

Creates/changes or checks the passcode entered by the user on the screen. The mode parameter indicates whether this callback method was called from a change passcode scenario, and if it was called for checking if the entered passcode matches the existing one, or it was called for storing the entered passcode as a new/changed passcode.

Parameters
passcode passcode entered by the user on the passcode screen
mode indicates if change scenario was used and if creation or checking has to be executed
fragment the caller fragment
Throws
PasscodeValidationException exception thrown if invalid passcode was entered
