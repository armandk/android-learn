maxCost

@NonNull fun maxCost(maxCost: Double): MemoryCache<K, V>

Enables the Cost Factor mechanism by providing the maximum cost allowed.

Note that the user must calls this method before providing the optional customized CacheCostFactor.

When the Cost Factor mechanism is enabled and if a customized CacheCostFactor is not specified, a default CacheCostFactor#onExceedMaxCost(List) method will be used, which will will provide the most costly cache entry to remove when a the aggregated costs during #put(Object, Object, double) exceeds the maximum costs.

Parameters

maxCost - Double: A double number > 0.

Exceptions

IllegalArgumentException - if maxCost <= 0

IllegalStateException - if #build() has been called

Return
MemoryCache<K, V>: this instance of memory cache.
