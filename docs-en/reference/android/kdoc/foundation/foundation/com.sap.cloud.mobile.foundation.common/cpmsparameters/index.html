CpmsParameters

open class CpmsParameters
Deprecated: As of release 2.2, use SettingsProvider to hold global reference for SettingsParameters Facilitates keeping a global reference to a SettingsParameters that has been initialized with all the settings-- HCPMS server URL, application ID, application version, and device ID.

Generally, the application developer will initialize the SettingsParameters once and get it from any where a oSettingsParameters is needed for the server communication.

The #init(SettingsParameters) must be called once and only once.
