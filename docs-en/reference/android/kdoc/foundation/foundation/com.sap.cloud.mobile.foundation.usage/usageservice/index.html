UsageService

class UsageService : MobileService

Represents the facade to initialize the usage feature.

To use this usage service, the basic step will be:

When using autoSession, the system will start a usage session when the application is brought to front, and end the session when on background, so the usage information of this session could be uploaded to server for analysis. If autoSession is false, then the client code can call recordSessionStart of this to start the usage session, then recordSessionEnd to end the session, then do the usage upload.

Client code can also call other helper methods of this class to log different usage information.
