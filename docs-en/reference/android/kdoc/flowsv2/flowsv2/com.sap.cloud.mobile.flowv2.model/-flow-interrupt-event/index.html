Represents the event that will somehow interrupt the current flow.

There are three types of such event, Error, Warning and Info.

Error happens when something really critical is encountered during the flow process, and the framework cannot handle the situation thus must terminate the flow. Then with a dialog showing the reason, the flow will be terminated after closing the dialog.

Info represents some information that should let user know but not interfere the flow process.

Warning means something bad happens but the flow can still proceed, but the user should be notified, so he can choose either 'Retry' or 'Continue'. For example, after authentication, if passcode policy cannot be retrieved from server, user can have a chance to go back and retry, or ignore for now and set passcode with default passcode policy.
