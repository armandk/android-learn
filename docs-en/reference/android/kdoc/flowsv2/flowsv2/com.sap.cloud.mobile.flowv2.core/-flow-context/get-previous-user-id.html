getPreviousUserId

fun getPreviousUserId(): String?
Content copied to clipboard

Retrieves the previous user id. For the very first onboarding flow, this will be 'null', then for later onboarding flows (create new account, or forgot passcode flows), this might be the same as that returned with getCurrentUserId if no user switch happens, or another user id if another user signs in or created.

This information will be available after FlowStateListener.onUserSwitched, then in callback functions like FlowStateListener.onOfflineEncryptionKeyReady (if com.sap.cloud.mobile.foundation.settings.SharedDeviceService) was added into com.sap.cloud.mobile.foundation.mobileservices.SDKInitializer) and FlowStateListener.onFlowFinished, it can be used to compare with getCurrentUserId to know whether a user switch happened or not.
