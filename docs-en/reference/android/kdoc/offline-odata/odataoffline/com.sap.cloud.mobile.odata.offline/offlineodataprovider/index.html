 fun sendEncryptedStore(newEncryptionKey: String!, successHandler: Action0!, failureHandler: Action1!): Unit

Sends the encrypted Offline OData store to sever. This feature offers a functionality to send the offline store which is encrypted to backend Server. The offline store files wiil be encrypted with temporary encryption key to avoid exposing original encryption key on the device. parameter ‘newEncryptionKey’ is always case sensitive, and they cannot contain leading or trailing spaces or semicolons. It is recommended to use ‘a-z’, ’A-Z’ and ‘0-9’ as new encryption key.

fun upload(categories: Array<OfflineODataRequestOptions.UploadCategory!>!, successHandler: Action0!, failureHandler: Action1!): Unit

Partial upload allows Offline OData applications to upload a subset of the requests in the request queue. Requests from multiple upload categories can be uploaded together by performing a partial upload, which is accomplished by calling the upload function with a list of upload categories to upload. All requests that the selected requests depend on are included in the upload, even if those requests are not in the specified upload categories.
