sendEncryptedStore

fun sendEncryptedStore(newEncryptionKey: String!, successHandler: Action0!, failureHandler: Action1!): Unit

Sends the encrypted Offline OData store to sever. This feature offers a functionality to send the offline store which is encrypted to backend Server. The offline store files wiil be encrypted with temporary encryption key to avoid exposing original encryption key on the device. parameter ‘newEncryptionKey’ is always case sensitive, and they cannot contain leading or trailing spaces or semicolons. It is recommended to use ‘a-z’, ’A-Z’ and ‘0-9’ as new encryption key.

Example Usage: 

// Send store asynchronously
      offlineODataProvider.sendEncryptedStore(newEncryptionKey, () -> logger.info("Send store completed successfully."),
                                     (error) -> logger.error("Exception encountered in offlineOData send store:" + error.getMessage()));

Parameters

newEncryptionKey - String!: the new encryption key can not contain special character like space and semicolon

successHandler - Action0!: a success callback function.

failureHandler - Action1!: a failure callback function.

note - a note which is not exceeding 480 characters long.
